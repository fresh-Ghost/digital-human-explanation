## 1. 产品概述 (Overview)

### 1.1 背景
传统的工业产品讲解依赖人工背诵固定讲稿，面临内容千篇一律、无法应对专业提问、人员培训成本高的问题。现有的数字人方案多侧重视觉形象（3D虚拟人），开发重、成本高，且往往忽视了“内容策展”和“即时互动”的核心需求。

### 1.2 核心价值
本产品定义为**“隐形的智能讲解中控”**。它移除具象化的虚拟人形象，专注于**听（需求理解）、想（策略生成）、说（自适应讲解）**。通过 Web 轻量化前端与硬件终端结合，实现“千人千面”的自动化讲解与专业问答。

### 1.3 用户角色
* **策展人 (Operator):** 负责配置讲解任务。通常是展厅管理员或销售经理。
* **观众 (Audience):** 现场听众。包括领导、客户、学生等。
* **系统 (Agent):** 包含“备课助手”和“现场讲解员”双重身份。

## 3. 非功能需求 (Non-functional Requirements)

* **响应延迟:** 语音打断延迟 < 1.5s；问答生成首字延迟 < 3s。
* **部署方式:** MVP 阶段支持公网 API 调用；后期支持 Docker 私有化部署。
* **硬件兼容:** 适配主流 Windows/Android 工控机，支持 USB 麦克风阵列。
* **数据安全:** 敏感参数库需具备访问控制权限。

## 5. 技术亮点 (Technical Highlights)

### 5.1 前端原生 WAV 编码
- 为解决 Windows 环境缺少 FFmpeg 导致的 ASR 兼容性问题，项目在前端实现了纯 JavaScript 的 WAV 编码器。
- 使用 Web Audio API 实时采样，手动构建 RIFF/fmt/data 头部，生成标准 16kHz/16bit 单声道音频流。

### 5.2 上下文感知 Prompt
- 在 System Prompt 中加入行为约束（禁止重复提问、自然交流、动态追问）。
- 利用智谱对话接口的 `messages` 列表（System + History + User）实现历史记忆。

### 5.3 汇总式脚本生成
- 在生成阶段将整个对话历史作为 `requirement` 输入 RAG 流程，提高生成脚本的针对性。
- 引入需求提取环节，过滤对话噪音，减少 RAG 检索干扰。

### 5.4 LangChain RAG 编排
- 使用 `langchain_community.vectorstores.Chroma` 管理知识库。
- 在生成讲解脚本或进行策展对话时，系统先通过 LangChain 进行检索，结合智谱 GLM-4-Flash 实现检索增强生成。
