## 1. 产品概述 (Overview)

### 1.1 背景
传统的工业产品讲解依赖人工背诵固定讲稿，面临内容千篇一律、无法应对专业提问、人员培训成本高的问题。现有的数字人方案多侧重视觉形象（3D虚拟人），开发重、成本高，且往往忽视了“内容策展”和“即时互动”的核心需求。

### 1.2 核心价值
本产品定义为**“隐形的智能讲解中控”**。它移除具象化的虚拟人形象，专注于**听（需求理解）、想（策略生成）、说（自适应讲解）**。通过 Web 轻量化前端与硬件终端结合，实现“千人千面”的自动化讲解与专业问答。

### 1.3 用户角色
* **策展人 (Operator):** 负责配置讲解任务。通常是展厅管理员或销售经理。
* **观众 (Audience):** 现场听众。包括领导、客户、学生等。
* **系统 (Agent):** 包含“备课助手”和“现场讲解员”双重身份。

## 2. 核心功能需求 (Functional Requirements)

### 2.1 模块一：智能知识库 (KMS)
**目标**：构建 AI 的“工业大脑”，确保回答准确、不出幻觉。

| ID | 功能点 | 描述 | 优先级 |
| :--- | :--- | :--- | :--- |
| **F-KMS-01** | **多模态素材管理** | 支持上传 PPT 图片、视频文件，并支持打标签（Tagging）。 | P0 |
| **F-KMS-02** | **非结构化数据清洗** | 支持导入 PDF/Word 手册，自动进行语义分块（Chunking）并向量化。 | P0 |
| **F-KMS-03** | **结构化参数库** | 提供表格录入界面，存储绝对参数（如：型号、转速、价格）。 | P1 |

### 2.2 模块二：交互式策展 (The Curator)
**目标**：通过对话，自动生成针对性的演讲脚本。

| ID | 功能点 | 描述 | 优先级 |
| :--- | :--- | :--- | :--- |
| **F-CUR-01** | **需求访谈 Agent** | 用户输入“给谁讲”，AI 自动追问（身份、时长、侧重点）。 | P0 |
| **F-CUR-02** | **策略生成** | AI 输出讲解策略摘要供确认（例：“策略：重商业价值，轻技术细节”）。 | P0 |
| **F-CUR-03** | **脚本生成与编辑** | 基于策略生成 JSON 格式的时间轴脚本，支持人工微调文案。 | P0 |

### 2.3 模块三：现场讲解终端 (The Runtime)
**目标**：执行讲解，响应打断。

| ID | 功能点 | 描述 | 优先级 |
| :--- | :--- | :--- | :--- |
| **F-RUN-01** | **轻量化播放器** | Web 端渲染图片/视频，根据脚本自动翻页、高亮局部。 | P0 |
| **F-RUN-02** | **语音合成 (TTS)** | 支持流式语音播放，预置 3 种风格音色（严肃、亲切、激情）。 | P0 |
| **F-RUN-03** | **智能打断 (Barge-in)** | 麦克风检测到用户人声提问时，毫秒级暂停当前播放。 | P0 |
| **F-RUN-04** | **RAG 问答** | 针对提问检索 KMS，生成语音回答。答完后询问是否继续。 | P0 |
| **F-RUN-05** | **状态可视化** | 屏幕显示波形动效（聆听中/思考中/讲解中），提供交互反馈。 | P1 |

## 3. 非功能需求 (Non-functional Requirements)

* **响应延迟:** 语音打断延迟 < 1.5s；问答生成首字延迟 < 3s。
* **部署方式:** MVP 阶段支持公网 API 调用；后期支持 Docker 私有化部署。
* **硬件兼容:** 适配主流 Windows/Android 工控机，支持 USB 麦克风阵列。
* **数据安全:** 敏感参数库需具备访问控制权限。
